# 实施记录

## 阶段 1：工程骨架

- TypeScript strict
- VS Code Extension API
- Activity Bar Webview View
- Extension/Webview 双入口 esbuild
- Vitest + ESLint + VS Code Electron
- VSIX 打包

主要文件：`package.json`、`esbuild.mjs`、`src/extension.ts`、`src/webview/main.ts`。

## 阶段 2：Provider 与本地数据

- 三家 Provider 默认配置和命令模板
- 跨平台 Shell 参数转义
- CLI 可执行文件探测与可运行验证
- Codex/Claude/Gemini 项目会话发现
- 聊天内容清洗、保留开场 + 最近消息
- 文件指纹缓存

主要文件：

- `src/core/providers.ts`
- `src/core/commandRenderer.ts`
- `src/services/providerDetection.ts`
- `src/services/sessionDiscovery.ts`
- `src/services/sessionContent.ts`

## 阶段 3：指标与套餐额度

- 最近 7 天 Token
- 历史 Token 总量
- 最近 7 天完成回合平均响应时间
- Codex app-server JSON-RPC 套餐额度
- Claude OAuth 套餐额度与安全 endpoint 约束
- workspace metrics cache

主要文件：

- `src/services/sessionMetrics.ts`
- `src/providers/codexUsage.ts`
- `src/providers/claudeUsage.ts`
- `src/services/providerUsage.ts`
- `src/services/projectState.ts`

## 阶段 4：Dashboard

- 搜索、Provider 筛选、计数、刷新
- 双指标卡片和 7 点平滑 SVG
- 灰/绿终端点与 Working/Ready
- Open/YOLO/Pin
- WeChat 风格聊天预览
- 紧凑 usage popup
- 响应式窄侧栏布局

主要文件：`src/webview/protocol.ts`、`src/webview/main.ts`、`src/webview/styles.ts`。

## 阶段 5：原生终端与实时状态

- Terminal Provider icon/name/cwd
- Shell Integration execute/read
- sendText + clipboard fallback
- Provider 历史增量活动监听
- Idle/Working/Ready reducer
- ANSI/控制序列与状态文本过滤
- `/exit` marker 与 shell execution end
- 多终端聚合、Tab close、Ready viewed

主要文件：`src/terminal/*`。

- 当前机器真实历史发现与指标基准
- npm 全依赖 0 漏洞
