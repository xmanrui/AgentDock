# 踩坑与平台差异

## 1. VS Code 不开放 Terminal Tab DOM

JetBrains 插件可以拿到 Terminal Content/Icon 绘制入口，因此能动态缩放 Provider Logo、画 Ready 绿角标，并把浮层锚定在标签上方。

VS Code 稳定 Extension API 只允许创建 Terminal、设置初始 `name/iconPath`、读取 Shell Integration 和监听生命周期，不允许：

- 修改 Terminal Tab DOM
- 动态替换/缩放标签图标
- 在图标上画角标
- 在 Workbench 任意坐标创建浮层

处理方式：原生 Terminal 保留静态 Provider Logo；Working/Ready 和滚动流式文本显示在 AgentDock 对应会话卡片。不要尝试注入 VS Code Workbench DOM，这既不稳定也不符合扩展安全模型。

## 2. Shell Integration 不是创建 Terminal 后立即可用

新 Terminal 的 `shellIntegration` 可能延迟出现。只在创建后检查一次会丢失输出。

处理方式：

1. 最多等待 2.5 秒 `onDidChangeTerminalShellIntegration`。
2. 可用时调用 `executeCommand()` 并立即 `read()`。
3. 不可用时降级 `sendText()`。
4. 继续监听 `onDidStartTerminalShellExecution`，Shell Integration 稍后识别命令时再接管输出。
5. 用 `WeakSet` 防止同一个 execution 被读取两次。

真实 Extension Host 测试第一次曾因单次输出时序失败，这套补偿后连续通过。

## 3. CLI 退出不等于 Terminal Tab 关闭

用户输入 `/exit` 后，Codex/Claude 进程结束，但 Shell Tab 仍停留在交互 shell。如果只监听 `onDidCloseTerminal`，卡片绿点会一直亮。

处理方式：

- 优先监听 `onDidEndTerminalShellExecution`。
- Unix 将命令包装为退出后写临时 marker，500ms 轮询作为双保险。
- Windows 不注入 Unix shell 语法，依赖 Shell Integration execution end；关闭 Tab 始终可清理。

## 4. IDE/VS Code 的 PATH 不等于用户登录 shell

从 Dock/GUI 启动时，经常找不到 NVM/FNM 安装的 CLI。更危险的是 PATH 中可能先出现一个存在但缺少平台 optional dependency 的 Node wrapper。

处理方式：搜索 PATH、常见目录、NVM/FNM/Volta/ASDF/Mise、macOS App Bundle 和 login shell；每个候选必须真实执行版本探测，失败后继续下一个候选。

## 5. Provider JSONL 可能包含超大工具结果

直接把“半行”无限累积会在工具输出达到数十 MiB 时占用大量内存，并阻塞后续 `task_complete`。

处理方式：增量监听使用 1 MiB 上限的 UTF-8 line decoder；超大行丢弃到下一个换行，后续生命周期记录继续解析。

## 6. Codex Token 是累计值，不可直接求和

`total_token_usage.total_tokens` 是累计值，重复记录和 CLI 重置都存在。

处理方式：正常增长取本次累计减上次累计；累计下降视为新一段，从当前累计重新计入；没有累计值时使用 `last_token_usage`。

## 7. Claude usage 消息会重复

同一个 Claude assistant message/request 可能多次写入，usage 还可能后续增大。

处理方式：按 message id，其次 request id 去重；同 key 保留 Token 更大的样本。Token 包含 input、output、cache creation 和 cache read。

## 8. Avg. Time 必须使用完成回合

不能用“最后更新时间减创建时间”或“用户消息到任意 assistant chunk”冒充一天平均响应。

处理方式：

- Codex：`task_complete.duration_ms`
- Claude：`system/subtype=turn_duration.durationMs`
- Gemini：用户消息到下一条 Gemini 消息的时间

仅统计有完成事件的回合，按完成日期求平均。

## 9. Webview popup 不能越出 Webview 边界

JetBrains 的 `JWindow` 可以弹到卡片左侧或面板上方；VS Code Webview 内元素会被 iframe 边界裁剪。

处理方式：优先按原方向定位；空间不足时在 Webview 内翻转位置。额度框保证不覆盖 Provider Logo，聊天框保持在视口内。弹框使用 `overflow: hidden + clip-path: inset(... round 8px)` 消除圆角外黑角。

## 10. 重渲染会让筛选按钮失焦

点击 Provider 同时会触发筛选重渲染，旧 DOM 被删除后 focus/hover 弹框会立刻进入隐藏计时。

处理方式：给 Provider button 写稳定 `data-provider-id`，重渲染后恢复同一按钮焦点并重新绑定 popup anchor。

## 11. 趋势图不能把参考数据点硬编码进去

样式参考和数据内容是两件事。趋势图固定读取最近 7 天，每天一个点；使用平滑 cubic path、圆角线帽、细点和轻渐变，不修改真实日数据。

## 12. 首次发现大量 Codex 历史

当前机器有 482 个 Codex rollout。读取每个文件到结尾会明显拖慢。

处理方式：存在 index 更新时间时，拿到 session meta 和第一条有效用户消息后立即停止读取；并发设置上限 512。界面先恢复 workspace state，扫描与指标在后台刷新。

## 13. 指标缓存不能只看 session id

历史文件会继续增长，日期跨天后 7 天窗口也会改变。

处理方式：持久缓存同时记录 source path 对应文件的 mtime、length 和 anchor date；三者一致才复用。多个指标一次批量写 workspace state，避免 N 次序列化。

## 14. npm 全局 cache 权限损坏

本机 `~/.npm/_cacache` 存在 root-owned 文件，普通 `npm install` 报 EACCES。

处理方式：不修改用户全局权限，使用 `--cache /tmp/agentdock-vscode-npm-cache` 完成安装。该问题与仓库代码无关。

## 15. 旧测试工具链有安全公告

初始 Vitest 2/Vite 和 esbuild 0.24 带来 6 个开发依赖公告，其中一个 critical。虽然不进入 VSIX，仍会影响本地/CI。

处理方式：升级 Vitest/coverage 4.1.10、esbuild 0.28.1、vsce 3.9.2；完整 `npm audit` 回到 0。

## 16. VS Code Electron 测试宿主解析 shell 很慢

隔离测试偶尔在 ptyHost 解析用户 login shell 时超时。

处理方式：测试启动加入 `--force-disable-user-env`，测试工作区指定 `/bin/sh` profile 且 `terminal.integrated.inheritEnv=false`。最终终端集成测试稳定返回 0。
